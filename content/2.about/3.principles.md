BluesMinds is designed around eight core principles that guide every architectural decision.

## 1. OpenAI Compatibility

BluesMinds is a **drop-in replacement** for OpenAI. Only `base_url` needs to change — your existing SDK code, request shapes, and response parsing all work unchanged.

```python
# Before (direct OpenAI)
client = OpenAI(api_key="sk-openai-key")

# After (via BluesMinds)
client = OpenAI(
    api_key="sk-bluesminds-key",
    base_url="https://api.bluesminds.com/v1"
)
```

## 2. Single Endpoint

One URL. All providers. All models.

```
https://api.bluesminds.com/v1
```

## 3. Automatic Failover

When a provider is degraded, BluesMinds silently reroutes your request to an alternate channel. No action required from you — your app always gets a response.

## 4. Cost Optimization

When multiple channels serve the same model, BluesMinds selects the most cost-effective option automatically.

## 5. Privacy by Default

Request and response bodies are **not stored by default**. Only metadata (latency, status codes, token counts) is recorded for billing. Opt-in to logging when you need it.

## 6. Rate Limits

| Plan | Requests/min |
|------|-------------|
| Free | 20 RPM |
| Trial / 10-Day / Unlimited | 15 RPM |
| Enterprise | Custom |

::callout{type="info" icon="lucide:info"}
On `429` errors, use jittered exponential backoff and respect the `Retry-After` header.
::

## 7. Explicit Model Selection

BluesMinds **never silently downgrades** your model. If a model is unavailable, you get a `404` — not a substitution.

## 8. API Separation

| Surface | Path | Authentication |
|---------|------|---------------|
| **LLM Inference** | `/v1/*` | `Bearer sk-...` |
| **Management** | `/api/*` | Session token |

::callout{type="warning" icon="lucide:alert-triangle"}
Never cross authentication methods between surfaces. LLM keys do not work on management endpoints and vice versa.
::
