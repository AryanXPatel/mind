Some models support web search to access real-time information beyond their training data.

## Models with Search

| Model | Search Type | Notes |
|-------|------------|-------|
| `grok-3` | **Native** | Built-in search, activates automatically |
| `gemini-2.0-flash` | **Google Search** | Grounding via `/v1beta/models` |
| Select variants | **Tool-based** | Requires your own search backend |

## Native Search

`grok-3` has built-in web search that activates automatically for time-sensitive queries. Results include source URLs.

```python
response = client.chat.completions.create(
    model="grok-3",
    messages=[{"role": "user", "content": "What happened in tech today?"}],
)
# Sources are included in the response
```

## Tool-Based Search

For models without built-in search, you can use a `web_search` function tool with external APIs like SerpAPI, Brave Search, or Bing:

```python
tools = [{
    "type": "function",
    "function": {
        "name": "web_search",
        "description": "Search the web for current information",
        "parameters": {
            "type": "object",
            "properties": {
                "query": {"type": "string"}
            },
            "required": ["query"]
        }
    }
}]
```

## Gemini Grounding

Google Search grounding is available via the beta endpoint:

```bash
POST https://api.bluesminds.com/v1beta/models/gemini-2.0-flash:generateContent
```

The standard `/v1/` endpoint uses tool-based supplementation instead.

## Best Practices

::callout{type="info" icon="lucide:lightbulb"}
- Use search for queries requiring data beyond training cutoffs
- Ask models to **cite source URLs** in their responses
- **Cache repeated queries** to reduce latency and cost
- Search calls **count against RPM limits**
::

## Limitations

- Native search is only available on specific models
- Tool-based search requires your own search API integration
- Long search results risk exceeding context window limits
