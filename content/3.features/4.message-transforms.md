BluesMinds automatically converts between message formats so your code only needs to use the **OpenAI-compatible structure**. Responses are always normalized back to OpenAI format.

## Supported Conversions

```
OpenAI format ──▶ Anthropic Messages API
OpenAI format ──▶ Google Gemini generateContent
OpenAI format ──▶ Any other provider format
```

## System Message Handling

Each provider handles system messages differently. BluesMinds normalizes this for you:

| Provider | Internal Handling | You Write |
|----------|------------------|-----------|
| **OpenAI** | Native `{"role": "system"}` | Standard OpenAI format |
| **Anthropic** | Top-level `"system"` field | Standard OpenAI format |
| **Gemini** | `system_instruction` mapping | Standard OpenAI format |

```python
# You always write this — BluesMinds handles the rest
messages = [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"},
]
```

## Native Anthropic Endpoint

For apps already built against the Anthropic SDK, BluesMinds also exposes a native endpoint:

```bash
POST https://api.bluesminds.com/v1/messages
```

This accepts Anthropic's native message format directly.

## Streaming

All streaming responses are normalized to **OpenAI's SSE format** regardless of the upstream provider:

```
data: {"choices":[{"delta":{"content":"Hello"}}]}
data: {"choices":[{"delta":{"content":" world"}}]}
data: [DONE]
```

## Limitations

::callout{type="warning" icon="lucide:alert-triangle"}
- **Audio STT** — Speech-to-text is provider-specific and not normalized
- **Provider-specific params** — Parameters like Anthropic's `top_k` are not forwarded through the OpenAI compat layer. Use the native endpoint if needed.
::
