BluesMinds is designed with privacy as the default. Your data is yours.

## Default Behavior

::callout{type="info" icon="lucide:shield-check"}
BluesMinds does **NOT** persist request or response body content by default. Only metadata is recorded.
::

**Metadata collected for billing:**
- Latency (ms)
- HTTP status codes
- Token counts (prompt + completion)
- Timestamps

## Opt-in Logging

Enable detailed logging via the Console or Management API when you need debugging, auditing, or request replay capabilities.

### Fetching Logs

```bash
# Step 1: Authenticate to get a session token
# Step 2: Query the log endpoint
curl https://api.bluesminds.com/api/log/ \
  -H "Authorization: <session_token>"
```

### Log Schema

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique log entry ID |
| `timestamp` | ISO 8601 | When the request occurred |
| `endpoint` | string | API path called |
| `model` | string | Model used |
| `usage` | object | `prompt_tokens`, `completion_tokens` |
| `latency_ms` | number | End-to-end latency |
| `status` | number | HTTP status code |
| `request_id` | string | Unique request identifier |
| `user_metadata` | object | Custom metadata you attached |

## Per-Request Metadata

Attach custom metadata to any request using the `user` field:

```python
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}],
    user="project:myapp,env:production"
)
```

This surfaces as `user_metadata` in your logs for filtering and analysis.

## Request Tracing

Capture the `X-Request-ID` response header or the `id` field from the response body for end-to-end request tracing.

```python
response = client.chat.completions.create(...)
request_id = response.id  # Use for support tickets
```

## Security Best Practices

::callout{type="warning" icon="lucide:alert-triangle"}
**Never** embed API keys in client-side code.
::

- Rotate keys regularly via Console or `POST /api/token/`
- Delete unused keys immediately
- Use **scoped keys** per project to contain potential compromise
- Store keys in environment variables or a secrets manager
