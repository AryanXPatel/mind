BluesMinds supports OpenAI-compatible tool/function calling for models that support it, including `gpt-4o`, `claude-sonnet-4-5`, and `gemini-2.0-flash`.

## Flow

::steps
### Send a request with `tools` defined

### Model replies with `finish_reason: "tool_calls"`

### Execute the function(s) locally

### Return results via `role: "tool"` messages

### Model produces the final answer
::

## Example

```python
import json
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-key",
    base_url="https://api.bluesminds.com/v1"
)

# Step 1: Define tools and send request
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get current weather for a city",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "City name"}
            },
            "required": ["city"]
        }
    }
}]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "What's the weather in Sydney?"}],
    tools=tools,
)

msg = response.choices[0].message

# Step 2: Check for tool calls
if msg.tool_calls:
    for tool_call in msg.tool_calls:
        args = json.loads(tool_call.function.arguments)
        # Execute your function
        result = get_weather(args["city"])

        # Step 3: Return results
        messages = [
            {"role": "user", "content": "What's the weather in Sydney?"},
            msg,
            {
                "role": "tool",
                "tool_call_id": tool_call.id,
                "content": json.dumps(result)
            }
        ]

    # Step 4: Get final answer
    final = client.chat.completions.create(
        model="gpt-4o",
        messages=messages,
        tools=tools,
    )
    print(final.choices[0].message.content)
```

## Key Notes

- **Arguments are JSON strings** — always parse with `json.loads()`, never use them directly
- **Parallel calls** — iterate all entries in `msg.tool_calls`; some models invoke multiple functions at once
- **`finish_reason: "stop"`** — the model answered directly without tools. This is valid behavior.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| No `tool_calls` in response | Verify model supports tools; validate your JSON Schema |
| `finish_reason` is `"stop"` | Model answered directly — this is valid |
| Arguments fail to parse | `arguments` is a JSON string; use `json.loads()` |
